# 법률문서 윤문가이드 (공개판)

법률서면 초안을 받아, **의미를 보존한 채 문장·표현·구성만** 다듬는다. 백지에서 새로 쓰는 것이 아니라, 이미 작성된 초안을 손보는 작업이다.

> **공개·배포 가능.** 이 스킬의 규칙과 예문은 국립국어원 공공 어문규범과 일반적으로 알려진 작법 지식에 근거해 **직접 작성한 오리지널 자료**입니다. 근거·배포 안내는 `references/SOURCES.md` 참조.

## 철칙 — 의미 불변

윤문은 **문체·리듬·표현·문장구성**만 손댄다. 아래는 **보존 대상으로 두고 바꾸지 않는다**:

- 법리·법적 주장·결론, 청구취지·청구원인의 내용
- 사실관계의 내용·순서·인과관계
- 날짜·금액·수치·비율·당사자명·고유명사
- 법령 조문·판례 번호·인용문(증거 표시 갑/을 제○호증 포함)
- 주장의 강도(확정/추정 구분), 긍정/부정·이중부정의 법적 의미

표현을 바꾸다 의미가 흔들릴 위험이 있으면 **고치지 말고 그대로 두고, 코멘트로만 제안**한다. 법리 오류가 의심되면 윤문하지 말고 사용자에게 알린다.

## 모드 결정 (Phase 0)

작업 시작 시 한 줄 출력: `legal-polish-public — {경량|정밀} 모드`

- **경량(기본)** — 문단·짧은 서면(대략 3,000자 이하). 탐지·윤문·자체검증을 한 패스로 처리.
- **정밀(`--정밀` 또는 제출용·장문)** — 탐지 → 윤문 → 충실도감사를 단계 분리. 사용자가 `--정밀`·"제출 전 검토"를 명시했거나 입력이 길거나(대략 8,000자 초과) 제출 임박 서면이면 자동 승급(1줄 고지).
- **범위 옵션** — 기본은 문장·표현만. `--구조`를 주면 단락 순서·논증 흐름·문서 종류별 점검 포인트(`references/doc-types.md`)까지 손본다.

입력 확보: 붙여넣은 텍스트를 그대로 쓰거나 파일 경로를 주면 Read로 읽는다. 원문은 작업 전에 그대로 보존하여 대조 가능하게 한다.

## 워크플로우

### 경량 모드
1. 입력 로드 + 모드/범위 1줄 고지
2. 관련 reference 로드 → `references/detection-checklist.md` 패스 0~A 전체(조사·호응 0 → 장문 1 → … → 췌언 5f → 어문규범 A)와 내용 플래그 6으로 위반 구간 식별(전수 표시 우선)
3. 의미 불변 원칙으로 문장·표현 윤문
4. 자체검증 6항 통과 확인
5. 출력: 수정전/후 대조 + 적용 원칙

### 정밀 모드 (탐지 → 윤문 → 충실도감사 분리)
1. **탐지(2회 독립 실행 → 결과 합치기)** — `references/detection-checklist.md`의 패스를 순서대로 돌려(전수 표시 → 개별 탈락 판단) 위반 구간을 카테고리·근거와 함께 목록화(아직 고치지 않음).
   - **탐지를 2회 독립으로 실행한다**(가능하면 별도 에이전트 2개 병렬, 아니면 순차 2회). 한 번의 탐지는 긴 문서에서 토큰을 끝까지 훑지 못해 회차마다 *다른* 항목을 놓치므로, 1회만으로는 빠짐이 생긴다.
   - 두 목록을 **하나로 합치되**, 같은 구간·같은 방향은 1건으로 **중복을 정리**한다. 한쪽만 잡은 항목을 서로 메워 누락이 크게 줄어드는 것이 두 결과를 합치는 핵심 이유다.
   - 합친 목록을 사용자에게 제시.
2. **윤문** — 목록을 따라 구간별로 수술적으로 고친다. 과윤문 금지(멀쩡한 문장은 두기).
3. **충실도감사** — 원문과 윤문본을 의미 단위로 대조해 아래를 전수 점검. 어긋나면 그 구간 **롤백**:
   - 법리·주장·결론 / 사실·날짜·수치·당사자·고유명사 / 법령·판례·증거표시·인용 / 주장의 강도·긍정·부정 의미
   - **장문이면 이 단계를 별도 에이전트로 돌리되, 원문 전체를 그대로 넘겨 비교한다**(요약본을 넘기면 오탐 발생).
   - 편집 주석은 윤문본 전문(서면 본문) **밖에만** 두고, 본문에 끼워 넣지 않는다.
4. **출력** — 수정전/후 대조 + 충실도감사 결과(이상 없음/롤백 N건) 보고.

### 자체검증 6항 (모든 모드 공통)
1. 의미 불변 철칙 위반이 없는가
2. 고친 문장이 원문보다 실제로 더 명확·간결한가(과윤문 아닌가)
3. 법률용어를 정확히 썼는가(쉬운 말로 바꾸다 법적 정확성을 잃지 않았는가)
4. 어문규범(`grammar-norms.md`)에 맞는가
5. 변경 근거를 원칙으로 설명할 수 있는가
6. 삭제한 내용이 정말 군더더기인가(요건사실·증거연결·논거가 아닌가)

## 출력 형식

수정전/후 대조표로 제시하고, 최종 윤문본 전문을 이어 붙인다. 파일 출력을 원하면 `{원파일명}_윤문.md`로 저장(원본은 덮어쓰지 않는다).

```
## 윤문 결과 — {모드}
### 주요 수정 (N건)
| # | 원칙 | 수정 전 | 수정 후 |
### 윤문본 전문
### 충실도감사 (정밀 모드): 의미 불변 점검 — 위반 N건 / 롤백 N건
```
