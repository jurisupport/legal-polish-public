# 탐지 패스 체크리스트 (커버리지 확보용)

> 이 파일은 principles.md·grammar-norms.md의 **규칙을 다시 쓰지 않는다.** 규칙은 그쪽에 있고, 여기서는 "그 규칙이 누락 없이 *결과로 출력되도록*" 강제하는 **기계적인 전수 점검 절차**만 정한다. 윤문 전, 아래 패스를 순서대로 1회씩 돌린 뒤 수정 목록을 확정한다. 목적은 모델의 주의력에 커버리지를 맡기지 않는 것이다.

원칙: **"열거 구조라 허용", "애매하면 보존"으로 후보를 조기 탈락시키지 않는다.** 먼저 전수로 *표시*하고, 탈락은 표시 후 개별 판단으로만 한다(표시 누락이 가장 큰 실패 모드다).

---

## 실행 규율 — 왜 길어지면 새는가 (반드시 먼저 읽는다)

지금까지 재현율이 낮았던 단일 원인: **문서가 길어질수록 패스를 문서 전체에 "한 번 훑고" 끝내서, 저빈도·후반부 항목을 통째로 놓쳤다.** 규칙이 없어서가 아니라(규칙은 grammar-norms·principles에 있다) 규칙을 *출력으로 끌어내는 전수 점검*을 끝까지 안 돌려서다. 그래서 모든 패스는 아래 두 규율을 강제한다.

1. **청크 강제.** 본문을 **항(項)·문단 단위로 분해**한 뒤, *각 청크마다* 패스 0~5f를 독립으로 1회씩 돌린다. "문서를 한 번 봤다"로 끝내지 않는다. 청크 N개면 각 패스를 N번 적용한다. 후반부 청크(결론·맺음말·진단서 인용 단락)가 가장 잘 새므로 마지막 청크까지 같은 강도로 적용한다.
2. **토큰은 끝까지.** 토큰 목록이 있는 패스(0·5·5b·5f·A)는 **검색 결과의 첫 건이 아니라 마지막 건까지** 처리한다. 같은 유형을 한 곳만 고치고 나머지를 빠뜨리는 *부분 적용*이 두 번째로 잦은 실패다.
3. **표시 ≠ 채택.** 전수 표시 → 개별 탈락 판단의 2단으로 돌린다. 표시 단계에서 "관용이라 됨/열거라 됨"을 쓰지 않는다.

> 자기점검: 수정 목록을 확정하기 전에 "패스 0·A·5f를 *각 청크마다* 돌렸는가, 아니면 문서를 한 번만 훑었는가"를 스스로 묻는다. 한 번만 훑었으면 재현율은 구조적으로 낮다.

## 패스 0 — 조사·호응(§3)  (최우선)

고빈도인데 그동안 패스가 없어 전량 누락됐던 유형이다. **각 청크에서** 다음을 본다.

- **목적격조사 오류 `~를/을` + 자동사·형용사류.** '동의·직면·봉착·접하다·일치·해당·반하다·관하다·대하다·이르다·임하다' 등은 `~에` 를 받는다. `증거를 동의합니다`→`증거에 동의합니다`, `위기를 직면`→`위기에 직면`. 특히 `동의` 는 `를/은/는 동의` 가 모두 오류이니 `~에 동의` 로 통일한다(`증거는 동의합니다`→`증거에 동의합니다`).
- **한 절에 목적격 `을/를` 중복.** 한 절(節) 안에 목적어가 둘 이상이면 앞엣것을 주제격 `은/는` 으로 분리한다(`선고기일을 충분한 기간을 두어 지정`→`선고기일은 충분한 기간을 두어 지정`).
- **주술 호응 끊김.** 주어와 서술어가 멀어져 호응이 깨졌는지, 주어가 증발했는지 본다(§3). 행위 주체가 모호하면 주어를 살린다.
- 의미 변동이 없는 조사·호응 교정이므로 **flag가 아니라 edit**.
- 흔한 누락: '증거를/는 모두 동의합니다' 가 의견요지·증거의견·결론에 세 번 나오면 **세 곳 모두** edit한다(한 곳만 고치지 않는다).

## 패스 1 — 장문(§2)

**각 청크에서** 문장 단위로 훑어 **글자수 ≥ 80자** 또는 **연결어미(-고/-며/-바/-여/-는바/-점,) 2회 이상**인 문장을 *전부* 분할 후보로 표시한다(임계를 낮췄다 — 이전 100자/3회는 만연체를 통과시켰다).

- 열거(①②③, 1)2)3))가 들어 있어도 *일단 표시한다.* "열거라서 길어도 된다"는 표시 단계에서 쓰지 않는다.
- **'~점, ~점 … 참작하여' 시루떡 문장**은 대표적 분할 대상이다: 결론을 먼저 밝히고 사유를 개조식(첫째·둘째…)으로 쪼갠다.
- **과거사실 + 장래예정이 한 문장**에 엉켰으면(`…입원하여 치료를 받았고, …회송될 예정입니다`) 시제 기준으로 두 문장으로 끊는다.
- 흔한 누락: 사실관계 서술 단락(입·퇴원 경과, 진단서 인용)의 한 문장에 경과·예정·평가가 여러 개 담긴 경우.

## 패스 2 — 따옴표(C-2)

본문의 **모든 `"…"` 를 1건씩** 직접인용 / 요약·간접인용으로 분류한다.

- 화자의 말을 토씨까지 그대로 옮긴 것만 큰따옴표 유지.
- 요약·간접·강조·용어 정의는 `'…'` 로 바꾸거나 따옴표를 제거한다.
- 흔한 누락: 상대방 주장을 요약해 큰따옴표로 감싼 구절을 인용으로 착각해 지나침.
- **단, 따옴표 *안*의 진술(피고인 진술 등)·공소사실 사실적시·날짜·병명·금액·사건번호는 표현조차 손대지 않는다**(보존 철칙).

## 패스 3 — 불필요한 복수 '들'(§4)

`들의 / 들과 / 들이 / 들을 / 들에` 를 전수 검색한다.

- 사람·기관 등 셀 수 있는 주체(원고들·피고들·증인들)는 보존.
- 무생물·추상명사(서버들의, 판결들과, 정보들이, 조치들을)는 군더더기이므로 '들' 제거 후보로 표시.

## 패스 4 — 지시어·표기·용어 통일

`본 N` 과 `이 N`(본 사건/이 사건, 본 준비서면 등) 혼용을 전수 검색한다.

- 한 문서 내 우세 표기로 통일하되, '본 준비서면' 같은 관용 표기는 예외로 둘 수 있다.
- **의미 변동이 없으므로 플래그가 아니라 edit로 실행한다.**
- 같은 패스에서 단위명사 불일치(서버 2,365개 vs 2,365대), 누락된 범위 표시(`6. 15.2024` → `6. 15.~2024`)도 함께 점검해 edit한다.

**4-A. 같은 대상을 가리키는 다른 명칭(동의어·서류명·전문용어) 통일.**
하나의 대상·서류·개념을 문서 안에서 서로 다른 말로 부르는 혼용을 *문서 전체를 가로질러* 전수로 표시한다.

- 서류명·용어 혼용: 같은 대상을 `제출`/`신청`(검사가 제출한 증거 ↔ 검사가 신청한 증거), `입퇴원확인서`/`입/퇴원확인서`, `요양급여의뢰서`/`진료의뢰서`처럼 달리 적은 경우 → 우세 표기 하나로 통일.
- **단, 의미가 정말 같다고 확신될 때만 edit한다.** 두 명칭이 *법적·의학적으로 다른 개념일 수 있으면*('대장암'과 '결장암', '마비성 장폐색'과 '난치성 장폐색', '해제'와 '해지') **통일하지 말고 패스 6 플래그로 넘긴다.**
- 흔한 누락: 소제목/캡션에서 쓴 명칭과 본문에서 쓴 명칭이 어긋나는 경우.

## 패스 5 — 명사화·번역투(§4)

다음 토큰을 **각 청크에서 전부 문자열 검색**한다(이 목록의 누락이 단일패스에서 가장 잦은 실패다 — 한 토큰이라도 빠뜨리면 그 유형 전체를 놓친다). **토큰을 확장했다.**

조사형 번역투:
`~에 대한 / ~에 대하여 / ~에 의하여 / ~에 의해 / ~에 있어 / ~에 있어서 / ~함에 있어 / ~으로 인한 / ~으로 인하여 / ~으로 인해 / ~를 통하여 / ~을 통하여 / ~를 통해 / ~을 통해 / ~에서의 / ~에서는의 / ~과의 / ~와의 / ~로서의 / ~로의 / ~에의`

명사화·동사화:
`진행하다 / 실시하다 / ~함이 상당 / 야기하다 / 필요가 발생 / 필요가 있 / 가지고 있 / 이루어지 / 주어지 / ~고 있다 / ~고 있습니다`

기타:
`~적(인/으로) / ~의 형태로 / ~하는 상황 / ~하는 방향으로`

- **`~에서의`·`~과의`·`~와의`·`~로서의`는 일본어 투의 명사 결합**이다 — 우리말 어순으로 푼다(`기관에서의 치료`→`기관에서 치료`, `변호인과의 접견`→`변호인과 접견`, `추적관찰이 예정되어`→`추적관찰을 받을 예정`). 이전 목록에 이 토큰들이 빠져 ~에서의·~과의를 통째로 놓쳤다.
- 명사화: '조사를 진행하다→조사하다', '입원치료의 필요가 발생하자→입원치료가 필요해지자', '회피의 의사를 가지고 있지 않고→회피할 의사가 없고', '치료가 이루어지지 않으면→치료받지 못하면', '배려가 주어지는 경우→배려해 주시면'.
- 번역투 조사: '~에 대한 + 명사'는 '~의'로, '~에 의하여/~에 의해 + 피동'은 능동으로, '~으로 인한/인하여/인해'는 '~에 따른/~ 때문에'로, '~를 통하여/통해'는 조사 '~로'로.
- **`~적`**: '정형외과적 평가→정형외과 평가', '현실적 가능성→현실적으로'처럼 굳어진 용어가 아니면 덜어낸다(단 '법률적·사회적' 등 관용은 강도 보고 판단).
- '~고 있다' 남용 절제: 진행·계속이 꼭 필요한 자리가 아니면 단순 시제로. 단 실제 진행 중인 상태(`현재 입원해 있다`)는 보존.
- 흔한 누락: 같은 유형을 한 곳만 고치고 나머지 구간을 빠뜨리는 부분 적용 → 검색 결과를 *끝까지* 훑는다(예: `~으로 인한`이 '신경뿌리병증으로 인한'·'디스크로 인한'·'거동장애로 인하여' 세 곳이면 세 곳 모두).

## 패스 5b — 어려운 한자투·평이화 어휘(§4 (e) 평이화 표)

다음 토큰을 **각 청크에서 전부 문자열 검색**한다. 누락이 가장 잦은 패스이므로 검색 결과를 한 건도 빠뜨리지 않는다.

- 부정 준말: `아니하` 를 어간째 검색해 **모든 활용형을 한꺼번에** 잡는다 — `아니한 / 아니함 / 아니하여 / 아니하나 / 아니할 / 아니하면 / 아니하고 / 아니하는 / 아니합니다` → '않-'으로.
- 예스러운 청원·한자어: `앙청 / 앙망 / 사료되 / 운위 / 자(者)로서` → 평이한 표현으로. 단 법률관행상 굳어진 표현은 강도 유지선에서 코멘트로만 제안할 수 있다.
- 가정·양보: `가사` → '설령/설사'.
- 한정: `에 한하여 / 에 한한` → '에만/~한 경우에만'.
- 근거: `에 기하여 / 에 기한` → '에 따라/에 근거한'.
- 불필요한 사동: `시켜 / 시키어 / 시킨 / 시킬 / 시키는 / 시키지 / 시켰` → 주체가 스스로 한 일이면 '하여/한/할/하는…'으로(`단절시킬 우려`→`끊을 우려`, `연동시켜`→`연동하여`). **'시킬·시키는'을 빠뜨리지 않는다**(어간 `시키` 로 검색).
- 한자 지칭·시점: `당해 / 동(同) / 익일 / 우(右)` → '그/같은/다음 날/위'.
- 모두 의미 불변이므로 **flag가 아니라 edit**. 단 인용 원문 안의 표현은 손대지 않는다.

## 패스 5c — 접속사 반복(§6)

`나아가 / 또한 / 그리고 / 그러나 / 따라서` 를 전수 검색해 **같은 접속사가 항·문단마다 반복**되는지, 그리고 **열거 안의 불필요한 접속사**를 본다.

- 같은 접속사가 누적되면 일부를 교체하거나 삭제.
- **열거 항목 사이의 군더더기 '그리고'**도 절제한다(`A와 B, 그리고 C에게`→`A와 B, C에게`).
- 의미 변동이 없으므로 edit로 실행한다(논리 강도는 유지).

## 패스 5d — 제목·구성(§8)

각 항목 **제목**을 1건씩 훑어, ① 제목이 막연한지, ② 인접 항목과 혼동되는지, ③ **제목이 서술문인지** 점검한다.

- **서술문 제목은 축약 문구로** 바꾼다(`판결선고시 법정구속을 면할 수 있게 해주시기 바랍니다.`→`판결 선고 시 법정구속 면제 요청`). 제목은 완결된 문장이 아니라 명사구가 적절하다.
- 막연하거나 옆 항목과 헷갈리는 제목은 그 항목의 결론·내용으로 구체화한다.
- 제목 구체화는 내용 추가가 아니라 **본문에 이미 있는 사실로 요약**하는 것이어야 한다.

## 패스 5e — 섹션 간 내용 중복(§1·§8)

같은 사실·주장이 **떨어진 항(項)에 거듭 진술**되는지 문서를 가로질러 점검한다.

- 같은 취지가 서로 다른 항(예: 4.라.항과 5.항, 본문과 결론부)에 중복되면 한 곳으로 통합하거나 뒤쪽을 짧은 참조로 줄인다.
- **단, 통합으로 사실·주장·강도가 손실되지 않을 때만 edit한다.** 두 진술이 미묘하게 다른 맥락(예: 같은 협진 사실이 한 곳은 '치료 필요', 한 곳은 '선고기일 지정 근거')이면 합치지 말고, 위험하면 "○항과 △항 통합 검토" 코멘트로만 제안한다.
- 흔한 누락: 결론부(맺음말)와 본문 쟁점항이 같은 사정을 두 번 말하는 경우.

## 패스 5f — 췌언·동어중복(§1 간결성)

군더더기·겹말·중복 추정어를 **각 청크에서 토큰 검색**으로 잡는다. 그동안 간결성 패스가 없어 이 유형을 통째로 놓쳤다.

- **중복 추정어**: `약 ~ 정도 / 약 ~ 가량 / 대략 ~ 정도` → 추정어 하나만(`약 2개월 정도`→`약 2개월` 또는 `2개월가량`). `기간이 소요` → `소요`.
- **겹말(동어반복)**: `매 ~ 마다`(`매 진료마다`→`진료 때마다`), `직후 + 즉각적`(`선고 직후의 즉각적인`→`선고 직후의`), `미리 + 예정`, `서로 + 상호`, `남은 + 여생`, `과반수 + 이상`.
- **의미중복 수식**: `너그럽게 + 선처`(선처=관대한 처분), `명백히 + 분명`. 단 강조 의도가 살아야 하면 코멘트로만.
- **불필요한 강조부사**: `매우 / 극히 / 실로 / 아주 / 너무나` — 법률문서 톤에서 덜어낸다(§1).
- **상투적 군더더기**: `~에 관하여 보건대`, `~한 사정이 있다고 보여지는 바`.
- ⚠ **강도 변화 주의**: 군더더기 제거가 *주장 강도를 올리거나 내리면*(예: '악화에 가까운'→'악화'는 강도 상승) 단정 수정하지 말고 코멘트로만. 순수 군더더기만 edit.

## 패스 A — 어문규범 띄어쓰기·붙여쓰기 전수 점검(grammar-norms 연동)

**grammar-norms.md의 규칙은 있는데 그걸 결과로 강제하는 전수 점검이 없어, 띄어쓰기 교정이 전량 누락됐다.** 이 패스가 grammar-norms를 *출력으로 끌어낸다*. 각 청크에서 아래를 전수 검색한다.

- **의존명사 띄움(grammar-norms A-1)**: `시 / 때 / 것 / 수 / 바 / 데 / 지(시간) / 만큼 / 뿐 / 대로 / 중 / 등`. 특히 **시(時)**: `선고시·판결선고시·도래시·신청시`→`선고 시·판결 선고 시·도래 시`(본문에서 반복되면 전부). `등`은 의존명사로 띄움.
- **접미사·합성어 붙임**: `가량`(`1주일 가량`→`1주일가량`), `여`(`10여 명`), `쯤`. 합성어 `그 동안`→`그동안`, `오래 전`→`오래전`, `한 데`→`한데`(grammar-norms A-6).
- **'제+숫자' 붙임, 단위명사 띄움**(A-2·A-3): `제 1심`→`제1심`, `3000만원`→`3000만 원`.
- **'이/그/저 + 명사' 띄움**(A-6): `이사건`→`이 사건`. **'및' 앞뒤 띄움, '안 되다' 띄움**(A-8).
- **되/돼, -로서/-로써, 율/률**(B-1) 혼동도 청크에서 함께 점검.
- 모두 의미 불변이므로 **edit**. 인용 원문·고유명사 표기는 예외.

## 패스 6 — 내용/논리 플래그(윤문 금지, 표시만)

날짜·수치·인과·선후관계의 **자체 모순**과 **원자료 확인 필요 항목**을 표시만 한다(고치지 않는다).

- 같은 사실의 날짜가 절마다 다른지, 선행 사건이 후행 날짜로 적혔는지.
- **날짜·범위 표기의 자릿수 탈락**: `2026. 5. .자`처럼 일(日)이 비었는지(앞 절과 대조해 탈락 의심 시 플래그, **임의 보충 금지**).
- **같은 증거번호 중복 부여**, 본문 증거표시와 입증방법 목록 불일치.
- **캡션 ↔ 본문 목록 대조**: 첨부서류·증거 목록에 있는데 본문에서 인용 위치·근거가 없는 항목(예: 진단서 2통 중 1통만 본문 인용 → 나머지 인용 여부 점검 플래그).
- **같은 실체를 다를 수 있는 전문용어로 기재**('대장암'/'결장암', '마비성'/'난치성' 장폐색): 표기 통일(4-A) 대상인지 개념 차이인지 불확실하면 **원자료 대조 필요 — 플래그만**.
- 금액 표기 방식 혼재, 당사자 표기 일관성, 입증방법에만 있고 본문 미연결 증거.
- 모순·불확실 의심 시 **고치지 말고 플래그만**(법리·사실이 걸리면 보존 우선).

## 패스 Z — 가벼운 토큰 점검(단일런 한정·선택)

> ⚠ **주의.** 이 패스를 무거운 "출현수=처리수" 대조표로 강제하면, 단일런에서 주의력이 토큰 표로 쏠려 **표에 없는 구조·명사화 항목을 거꾸로 흘리는** 역효과가 난다. 토큰 잔존을 0으로 만드는 일은 패스 Z가 아니라 **정밀 모드의 2회 독립 탐지 → 결과 합치기**로 달성한다. 단일런(경량 모드)에서만, 시간이 남을 때 *가볍게* 아래를 훑는다 — 강제 대조는 하지 않는다.

확정 직전, 아래 토큰만 빠르게 한 번 더 본다(없으면 통과):
`통하여 · ~으로 인한/인하여 · ~에서의/~과의/~와의 · 매~마다 · 그 동안 · 시키-(단절시킬) · 의존명사 시(판결선고시) · 가량 · 형태로 · 아니하- · 증거+동의 · 앙청/가사`

- 보존(강도·법리 우려로 안 고침)도 정당하다. 누락(아예 못 봄)과 구별해 사유만 1줄 남긴다.

### 실행 순서 요약

**[청크 분해]** → 각 청크마다: 0(조사·호응) → 1(장문) → 2(따옴표) → 3(복수 들) → 4(표기·용어 통일) → 5(명사화·번역투) → 5b(평이화 어휘) → 5c(접속사) → 5d(제목) → 5e(섹션 중복) → 5f(췌언·중복) → A(어문규범 띄어쓰기) → **[문서 전체]** 6(내용 플래그) → Z(가벼운 토큰 점검 — 단일런 한정·선택).

> **재현율의 진짜 지렛대 = 단일런 강화가 아니라 2회 독립 탐지 → 결과 합치기**(정밀 모드). 단일런은 회차마다 다른 항목을 놓치므로, 같은 패스를 2번 독립으로 돌려 결과를 합치면 누락이 크게 준다. 패스 Z를 무겁게 만들어 한 런에 다 담으려 하지 말 것 — 주의력 재분배로 역효과다.

- 0·1·5·5b·5c·5d·5e·5f·A = **edit**, 4 = 의미 같으면 edit, 6 = **flag**.
- 각 패스는 *전수 표시 → 개별 탈락 판단*의 2단.

> **재현율을 떨어뜨리는 3대 실패 모드 — 매번 자기점검한다.**
> 1. **문서를 한 번만 훑음** → 청크별 반복(실행 규율 1)으로 막는다. 후반부 청크가 가장 잘 샌다.
> 2. **저빈도·1회성 항목 누락** → ~에서의·~과의·~적·필요가 발생·시킬·매~마다·판결선고시처럼 한 번만 나오는 토큰을 패스 0·5·5b·5f·A 토큰 목록 *끝까지* 검색.
> 3. **부분 적용** → 같은 유형 첫 건만 고치고 나머지를 빠뜨림. '증거를 동의'(3곳), '~으로 인한'(3곳), '아니하-'(다수)는 *전 구간* edit.
